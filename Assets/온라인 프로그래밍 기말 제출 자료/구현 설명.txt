게임 유저 데이터 관리 및 상점/인벤토리 시스템 구현 설명서
1. 개요
본 프로젝트는 Unity와 Firebase Realtime Database를 연동하여, 유저의 계정 정보, 게임 내 재화(Coin), 인벤토리(아이템 보유), 및 유닛 보유 상태를 실시간으로 동기화하고 관리하는 시스템을 구현함.

2. 주요 클래스 및 기능 구조
1) 데이터 모델링 (UserData.cs)
역할: 유저 데이터의 초기 구조를 정의함.

주요 항목: 닉네임, 코인, 점수와 더불어, Dictionary 구조로 관리되는 UnitList와 Inventory를 포함함.

특징: Newtonsoft.Json을 사용하여 복잡한 자료형을 JSON 문자열로 직렬화하여 데이터베이스에 저장함.

2) 로그인 시스템 (UserLogin.cs)
역할: Firebase 데이터베이스의 UserInfo 경로를 탐색하여 입력된 닉네임과 일치하는 유저를 검증함.

인증 유지: 로그인 성공 시 PlayerPrefs에 UserKey를 저장하여, 앱 전역에서 유저 고유 식별자로 활용함.

3) 상점 및 인벤토리 관리 (ShopManager.cs, UnitShopManager.cs, InventoryManager.cs)
구매 로직: 재화(Coin)를 확인하여 구매 가능 여부를 판단하고, UpdateChildrenAsync를 통해 서버의 코인과 아이템/유닛 보유 데이터를 일괄 업데이트함.

인벤토리 조회 및 사용: 서버에서 Inventory JSON 데이터를 가져와 객체로 복원(Deserialize)하며, 아이템 사용 시 수량을 차감하고 실시간으로 저장함.

3. 핵심 기술 요소
Firebase Realtime Database: 비동기 네트워크 통신을 통한 실시간 데이터 동기화.

JSON 직렬화/역직렬화: Newtonsoft.Json 라이브러리를 활용해 Dictionary와 같은 복합 컬렉션 데이터를 원격 데이터베이스와 효율적으로 주고받음.

스레드 동기화: UnityMainThreadDispatcher를 사용하여 비동기 네트워크 작업의 결과를 안전하게 메인 스레드(UI)에 반영함.

데이터 유지: PlayerPrefs를 사용하여 로컬 환경에서 유저 세션을 관리함.

4. 구현상의 특징
확장성: 아이템과 유닛을 Dictionary 구조로 관리하여, 새로운 아이템이나 유닛이 추가되어도 코드 수정 없이 데이터 확장 가능함.

사용자 피드백: 모든 통신 과정에서 MessageText를 UI에 출력하여 현재 데이터 로딩 상태나 구매 성공/실패 여부를 유저가 즉시 확인할 수 있도록 설계함.
